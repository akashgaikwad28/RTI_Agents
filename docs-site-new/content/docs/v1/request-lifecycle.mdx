# Request Lifecycle

Every citizen query flows through a rigorous sequence of routing, verification, and formatting nodes governed by LangGraph.

## Flow of Execution

1. **Citizen Query (`POST /submit`)**: The user submits a query. JWT validation ensures authentication.
2. **LangGraph Router**: Inspects the intent.
3. **Classification**: Identifies the correct government department.
4. **Retrieval**: Performs multilingual RAG search to fetch relevant acts and policies from FAISS.
5. **Debate Phase**: A `CriticAgent` evaluates RAG contexts, while a `DefenderAgent` argues validity. A `VerifierAgent` cross-references citations, and a `ConsensusEngine` aggregates insights.
6. **Human-In-The-Loop (HITL) Pause**: The graph interrupts execution. An admin is notified via email.
7. **Formatting**: Legal tone is applied based on admin feedback.
8. **Response & Memory**: The request is tracked, and the final trace is stored.
